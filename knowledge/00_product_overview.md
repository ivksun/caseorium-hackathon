# обзор продуктовой линейки Just AI

> контекст: Just AI -- российский вендор AI-решений для enterprise.
> основной фокус 2026 -- агентская платформа как единый стек для GenAI-трансформации.
> все продукты ниже -- части единой экосистемы, но некоторые (Jay Guard) продаются и отдельно.

---

## продукт 1: Just AI Agent Platform

### 1. что это
единая платформа для автоматизации процессов и коммуникаций с помощью AI-агентов.
включает: no-code конструктор workflow и диалогов, pro-code IDE (JS, DSL, Python), каталог готовых шаблонов, аналитику и мониторинг, корпоративную безопасность (SSO, RBAC, логирование, шифрование).
в Q1 2026 -- AI-ассистент для разработки (генерация процессов из текстового описания, автотесты, автоисправление ошибок).

### 2. какую задачу решает
заменяет зоопарк разрозненных сервисов единым стеком для всех типов AI-автоматизаций:
- внешние коммуникации (клиентский сервис, лидогенерация)
- внутренние процессы (HR, закупки, бэк-офис)
- кросс-процессы (начинаются снаружи, проходят внутри, возвращаются наружу)

### 3. для кого
- enterprise с оборотом от 3 млрд руб. и выше (основной фокус -- top-10 enterprise с оборотом >100 млрд)
- роли: CEO, CDO/CTO, руководители направлений, руководители проектов
- отрасли приоритета: банки и финтех, телеком, ритейл, промышленность (металлургия, добыча), страхование
- вторичная аудитория: разработчики/проджекты внутри enterprise, которые ищут инструмент для прототипирования агентов (далее масштабирование на всю компанию)

### 4. где неуместен
- малый бизнес (до 500 млн оборота) без сложных процессов -- overkill
- компании, которым нужен только чат-бот без интеграций -- слишком тяжёлый стек
- сценарии, где нет регуляторных требований к данным и достаточно SaaS-решений типа ChatGPT напрямую
- бизнесовые пользователи без IT-команды, которые не готовы к внедрению платформы (им скорее Jay Copilot или готовые SaaS-решения)

### 5. реальные отличия от конкурентов
- **мульти-LLM**: подключение облачных и локальных моделей (GigaChat, YandexGPT, OpenAI, Anthropic, Gemini, on-prem LLM) в одном контуре
- **гибридные сценарии**: комбинация LLM и NLU в одном агенте -- ускорение работы и снижение расхода токенов (у конкурентов обычно или LLM, или NLU)
- **no-code + pro-code**: MVP за минуты в визуальном редакторе, расширение кодом на JS/Python/DSL (конкуренты -- обычно только одно из двух)
- **Jay Guard встроен**: контроль данных при работе с LLM на уровне платформы (маскирование, блокировка, защита от prompt injection)
- **30+ готовых интеграций**: RAG (Jay Knowledge Hub), CRM, ERP, PMS, телефония, мессенджеры, соцсети, email, голосовые каналы, MCP
- **FinOps из коробки**: лимиты по токенам, минутам, каналам, алерты и дашборды -- контроль GenAI-бюджета
- **voice-технологии**: собственные ASR/TTS + выбор вендоров (SpeechKit, T-Bank VoiceKit, 3iTech, Salute, Elevenlabs)
- **deployment**: облако Just AI или on-prem в контуре заказчика
- **NLU-стек CAILA**: собственные NLP/NLU технологии, не зависящие от сторонних LLM

## продукт 2: Jay Copilot

### 1. что это
корпоративная AI-платформа, предоставляющая сотрудникам доступ к множеству LLM через единый интерфейс: web, Telegram, расширение браузера.
поддерживаемые модели: GigaChat, YandexGPT, ChatGPT (GPT-4), Grok, DeepSeek, Claude, Gemini, Qwen, DALL-E, Flux.
включает: AI-базы знаний (RAG), корпоративные AI-приложения (готовые промпты для задач), аналитику запросов сотрудников.

### 2. какую задачу решает
- контролируемый корпоративный доступ к GenAI вместо хаотичного использования сотрудниками ChatGPT через VPN
- +30% к эффективности сотрудников за счёт автоматизации рутины: тексты, изображения, видео, анализ данных, ресёрч сайтов
- подключение корпоративной базы знаний -- ответы по внутренним документам (XLS, аудио, видео, SQL, СУБД)
- сбор реальных сценариев использования AI сотрудниками для масштабирования лучших кейсов на всю компанию

### 3. для кого
- любая компания от среднего бизнеса, которая хочет дать сотрудникам доступ к LLM под контролем
- роли пользователей: маркетологи, копирайтеры, дизайнеры, аналитики, HR, рекрутеры, разработчики, юристы, продакт-менеджеры
- роль закупщика: CDO, CTO, руководитель IT

### 4. где неуместен
- компании, которым нужна глубокая автоматизация процессов (нужна Agent Platform)
- сценарии, требующие кастомных интеграций с бизнес-системами (CRM, ERP)
- компании с жёстким запретом на облачные LLM и без готовности к on-prem (хотя Jay Copilot поддерживает on-prem LLM)

### 5. реальные отличия от конкурентов
- **работает в России без VPN** (в отличие от прямого ChatGPT)
- **мульти-LLM в одном интерфейсе**: можно тестировать разные модели на одной задаче, выбирать лучшую
- **AI-приложения**: готовые промпт-шаблоны, не требующие навыков промптинга (кнопка "сделай лучше" автоулучшает запрос)
- **корпоративная безопасность**: интеграция с Jay Guard для маскирования/блокировки чувствительных данных
- **аналитика использования**: видно, что сотрудники спрашивают, какие кейсы востребованы -- основа для масштабирования
- **RAG по корпоративным документам**: загрузка внутренних материалов, ответы по ним в интерфейсе
- входит в **реестр отечественного ПО**

## продукт 3: Jay Guard

### 1. что это
модуль контроля данных при использовании LLM. шлюз между пользователем/агентом и нейросетью, который анализирует все запросы и ответы, маскирует или блокирует чувствительные данные, защищает от prompt injection и jailbreak.

### 2. какую задачу решает
- минимизация рисков утечки ПДн и коммерческой тайны при работе с облачными LLM
- маскирование данных с сохранением семантики запроса (замена на вымышленные данные)
- фильтрация и цензурирование контента по заданным темам
- защита от prompt-based атак (injection, jailbreak)
- централизованное применение политик безопасности AI
- ведение журнала всех запросов и ответов, интеграция с корпоративными системами оповещений

### 3. для кого
- **основной сегмент**: крупный регулируемый enterprise -- банки, финтех, телеком, промышленность, госсектор, компании с госучастием
- **вторичный сегмент**: компании с внутренними AI-платформами (технологические компании, интеграторы, вендоры)
- **третичный сегмент**: средний бизнес на этапе масштабирования AI (от пилотов к production)
- роли: CISO, руководитель ИБ, CDO/CTO

### 4. где неуместен
- малый бизнес и стартапы без регуляторных рисков
- индивидуальные пользователи
- компании, использующие AI только в нерабочих сценариях
- ситуации, где достаточно классического DLP (без AI-специфики)

### 5. реальные отличия от конкурентов
- **на рынке РФ всего 2 прямых конкурента** (HiveTrace, InFera) -- оба заметно менее зрелые
- **не DLP, а AI-специфичный контроль**: классические DLP закрывают трансграничную передачу, но не контролируют что и как генерирует AI
- **маскирование с сохранением семантики** (а не просто блокировка)
- **гибкая архитектура**: cloud + on-prem, работа по протоколу OpenAI API
- **поддержка кастомных фильтров**: не только предустановленные, но и свои правила + ML NER
- **часть экосистемы Agent Platform**, но может покупаться отдельно
- **эволюция в 2026**: от маскирования ПДн к управлению поведением LLM, защите AI-агентов и цепочек действий
- входит в **реестр отечественного ПО**

## продукт 4: Jay Knowledge Hub

### 1. что это
RAG-движок для обучения AI-агентов на корпоративных документах без ручной разметки. часть экосистемы Agent Platform.

### 2. какую задачу решает
- превращение корпоративной документации в базу знаний для AI-агентов и Jay Copilot
- ответы на вопросы по внутренним документам (инструкции, регламенты, FAQ, техдокументация)
- поддержка форматов: XLS, аудио, видео, SQL, СУБД

### 3. для кого
- enterprise, использующий Agent Platform или Jay Copilot
- команды поддержки, HR, operations -- все, кому нужно быстро находить информацию в корпоративных документах

### 4. где неуместен
- компании без структурированной документации
- сценарии, где достаточно полнотекстового поиска без генерации ответов

### 5. реальные отличия от конкурентов
- нативная интеграция с Agent Platform и Jay Copilot (не требует отдельной настройки RAG-pipeline)
- поддержка мультимодальных форматов (аудио, видео, не только текст)
- работает как с облачными, так и с on-prem LLM

## продукт 5: CAILA

### 1. что это
ML-платформа компании Just AI: NLP/NLU технологии, инференс LLM, собственные языковые модели. инфраструктурный слой внутри Agent Platform.

### 2. какую задачу решает
- обработка естественного языка без зависимости от сторонних LLM
- гибридные сценарии: NLU для простых интентов (быстро, дёшево) + LLM для сложных (глубоко, дорого)
- снижение расхода токенов за счёт маршрутизации запросов

### 3. для кого
- команды разработки внутри enterprise, строящие AI-решения на Agent Platform
- интеграторы и вендоры, встраивающие NLU в свои продукты

### 4. где неуместен
- проекты, где достаточно только LLM без классического NLU
- компании без технической команды

### 5. реальные отличия от конкурентов
- собственный NLU-стек, а не обёртка над OpenAI/GigaChat
- возможность гибридной работы LLM + NLU в одном pipeline (уникально для российского рынка)
